格式说明：
/name METHOD
解释1 para1 type1
解释2 para2 type2
RET:
"A"
含义
代表访问路径为 /api/name，方法为 METHOD，一共需要两个 Http 参数，其中第一个参数类型为 type1，第二个参数类型为 type2，返回值为 type2，它们的含义分别为解释1和解释2，该 API 的返回值为 "A"，含义为含义
example：
/test POST
用户身份 cookie string
RET:
"Male"/"Female"
用户性别。

正文：
/verifycookie GET
用户身份 cookie string
RET:
"Y"/"N"
成功返回 Y，失败返回 N

/login GET
用户名 usrname string
密码 paswd string
RET:
["Y"/"N",cookie/error]
成功返回 ["Y","..."] 表示用户身份信息（需要保存到浏览器 cookie 中），失败返回 ["N","..."] 表示失败原因

/updinfo GET
用户身份 cookie string
新的用户名 usrname string
新的密码 paswd string
新的公开代码状态 pubcode string
RET:
"Y"/"N"
更新操作是否成功
